# /tako-list — 자연어 → JQL 필터링

이 커맨드의 가치: **사용자 자연어 → JQL 인자 매핑**. 셸 `tako list` 의 인자 형태로 변환해 호출하고, 결과 표만 깔끔히 노출한다.

## 작동 환경 가정

- `tako init` 완료, 본인 권한으로 검색 가능 상태
- 사용자 의도는 *조회* (생성·수정 아님)

## 흐름

### 1. 자연어 → 인자 매핑

사용자 발화에서 다음 인자로 추출:

| 사용자 표현 | 인자 |
|---|---|
| "내 티켓" / "내가 한 거" / "맡은 거" | `--assignee me` |
| "이번 주" / "최근 일주일" | `--updated 7d` (또는 1w) |
| "어제 / 24시간" | `--updated 1d` (또는 24h) |
| "지난 한 달" / "최근 30일" | `--updated 30d` |
| "이번 주에 *만든* / 최근에 *생성된*" | `--created 7d` (만들어진 시점 기준) |
| "5월에 만든 거" | `--created 2026-05-01` |
| "기한 지난 / 마감 넘긴" | `--due overdue` |
| "기한 없는" | `--due none` |
| "6월 15일까지" / "기한 6월 안" | `--due "<=2026-06-15"` |
| "SP 3 이상" / "포인트 큰 거" | `--sp ">=3"` |
| "SP 미정 / 추정 안 된" | `--sp none` |
| "전부 / 다 보여줘 / 페이지 끝까지" | `--all` (모든 페이지 자동, 페이지당 100 max) |
| "진행중" / "검토대기" / 상태 이름 | `--status <이름>` (여러 개면 반복) |
| "에픽만" / "버그수정만" | `--type <이름>` (여러 개면 반복) |
| "WL-9200 자식" / "부모가 WL-9200" | `--parent WL-9200` |
| "라벨 foo" / "foo 라벨" | `--label foo` |
| "WL / ABC 프로젝트" / "여러 프로젝트" | `--project WL --project ABC` (반복) |
| "정렬 관련" / "결제 텍스트" | `--query 정렬` |
| "최대 50개" | `--limit 50` |
| "엑셀로 / CSV 로 저장 / 내보내기" | `--csv --output <파일경로>` (사용자가 경로 명시 안 하면 `tako-list-YYYY-MM-DD.csv` 같이 적당히 제안) |

복합 표현이면 *AND* 로 합쳐 인자 여러 개. 사용자가 *원하는 인자를 정확히* 지정한 경우 (예: `/tako-list --jql "..."`) 그대로 통과.

### 2. 셸 호출

```bash
tako list \
  --assignee me \
  --status 진행중 \
  --updated 7d \
  --limit 20
```

`--json` 은 *LLM 후속 분석*이 필요할 때만. 보통 사람 친화 표 출력이라 *생략*.

### 3. 결과 노출

stderr 의 `JQL:` 라인은 *디버깅용*이라 사용자에게 그대로 노출. 사용자가 *예상 외 결과* 일 때 JQL 보고 자기 의도 다시 확인 가능.

stdout 의 표를 그대로 사용자에게 보여준다. 결과 0건이면 "(결과 없음)" 만 stderr.

### 4. 후속 요청

- "더 보고 싶어" → `--limit` 늘려 재호출
- "WL-8876 자세히" → `/tako-check WL-8876` 또는 `tako show` 흐름 안내
- "방금 거 중에 진행중만" → 직전 인자에 `--status 진행중` 추가해 재호출

## 의사결정 사전 상의

추정하지 말고 사용자에게 묻기:

- **담당자 표현이 *한국어 이름*** (예: "주열이가 한 거", "강민성") — v1.x 는 한국어 이름 매칭 미지원. 사용자에게 *이메일 또는 accountId* 받기.
- **상태 이름이 모호** — "끝난 거" 가 *완료* 인지 *해결* 인지 *닫힘* 인지. 가능한 상태 목록 알려달라거나 본인 환경 가서 정확한 이름 확인 요청.
- **결과가 0건** — 조건 너무 좁거나 잘못된 이름. 한 단계 풀어보거나 `tako list --jql "..."` 직접 작성 안내.

## 실패 처리

- `assignee 값 형식 미지원` — 한국어 이름 입력 시. 사용자에게 안내.
- `상태/유형이 존재하지 않음` (Jira 400/422) — 응답 body 그대로 보여주고 정확한 이름 확인 요청.
- `--jql` 직접 작성 권유 — 표준 인자로 표현 안 되는 케이스.

## 자연어 매핑 — 주의할 점

- *추정* 위험: "내 티켓" 은 `me` 명확. 단 "최근" 은 일주일인지 한 달인지 모호. 모호하면 *짧게 한 번 확인* (예: "최근'이 며칠 정도일까요? 보통 7d / 30d 중 선택").
- 한국어 상태·유형 이름은 *Jira 에 정의된 정확한 이름* 그대로 써야. 사용자가 *느슨한 표현* 쓰면 짚어주고 정확한 이름 받기.
- `tako list` 의 `--limit` 기본은 20. 사용자가 "다 보여줘" 같이 말해도 *50/100 정도*로 명시 한도 두기 (한 번에 1000개 같은 건 위험).
