# /tako-check — 세션 작업 ↔ 티켓 명세 검토

이 커맨드의 가치는 한 가지: **현재 세션에서 진행한 작업이 지정 티켓의 요구사항을 얼마나 충족하는지 LLM 이 대조해서 보고**한다. 티켓 조회 자체는 `tako show` 셸 명령이 책임진다.

## 작동 환경 가정

- `tako init` 으로 설정·인증 완료된 상태
- 사용자가 *이미 진행한 작업*이 세션 컨텍스트에 있어야 의미 있음 (빈 세션이면 가치 거의 없음 — 사용자에게 한 번 안내)

## 흐름

### 1. 인자 정규화

사용자가 준 인자에서 이슈 키 추출:
- `WL-8876` 그대로
- `https://<site>/browse/WL-8876` URL 이면 키만 뽑기
- 형태 안 맞으면 사용자에게 다시 받기

### 2. 티켓 조회

```bash
tako show <KEY> --json
```

`--json` 으로 받아 LLM 이 파싱. 응답 구조: `{"issue": {...}, "comments": [...], "url": "..."}`. 셸 명령이 인증·REST·코멘트 조회·ADF 변환까지 다 처리하므로 본문이 별도 처리할 것 없음.

조회 실패(404, 401, 403)면 셸 명령이 한국어 에러를 stderr 로 출력함 — 그대로 사용자에게 노출하고 흐름 중단.

### 3. 세션 작업 요약

세션 컨텍스트에서 *사용자가 실제로 무엇을 했는지* 한국어로 추출. 다음 정도가 적당하다:

- 변경한 파일 / 추가·수정한 함수 (있으면)
- 분석·디버깅 흐름 (코드 변경 없이 분석만 했다면)
- 의사결정 / 사용자 합의 사항

세션이 *애매하거나 비어 있으면* 진행하지 말고 사용자에게 한 번 묻는다.

### 4. 대조 + 보고

티켓 명세 (특히 *설명* / *완료 조건* / *최근 코멘트*) 와 세션 작업을 항목별로 대조. 보고 형식:

```
[KEY] 유형 — 제목
링크: https://<site>/browse/<KEY>

## 티켓 요구사항 (요약)
- 항목 1
- 항목 2
- 항목 3

## 세션에서 한 작업 (요약)
- 변경 / 분석 / 결정 한 줄씩

## 대조
[OK]   <충족한 항목 + 근거 (파일:라인 등)>
[부분] <부분 충족 + 무엇이 부족>
[누락] <명세에 있는데 세션에 없음>
[제안] <세션에 있는데 명세에 없음 / 보완 제안>

## 결론
한두 줄로 종합 판단.
```

### 5. 마커 사용 규칙

- `[OK]` — 명세 항목이 세션 작업으로 *명확히* 처리됨 (근거 가능할 때만)
- `[부분]` — 일부 처리, 일부 누락. 무엇이 부족한지 명시
- `[누락]` — 세션에 흔적 없음. 본인이 빠뜨렸을 가능성
- `[제안]` — 세션에 있는데 명세 외 항목. 또는 추가 권장 (테스트 등)

이모지 안 씀.

## 의사결정 사전 상의 (반드시)

다음 상황에서는 LLM 이 추정하지 말고 사용자에게 먼저 묻는다:

- **세션 컨텍스트가 비어 있거나 매우 짧음** — "검토할 작업 흐름이 세션에 보이지 않습니다. 검토할 영역을 알려주시거나 더 작업하신 후 다시 호출해 주세요."
- **티켓 명세가 모호** — 설명이 짧거나 완료 조건 없음. "티켓 명세가 모호해 대조 기준이 약합니다. 그래도 진행할까요?"
- **세션 갈래가 여럿** — 한 세션에 *여러 작업*이 섞여 있음. 어느 흐름을 티켓 대상으로 볼지 한 번 확인
- **사용자가 코드 변경 없이 *분석만* 한 경우** — `[OK]` 마커를 함부로 쓰지 말 것. 코드 근거가 없으면 `[부분]` 또는 `[제안]` 으로 표기

## 후속 요청

- "특정 항목 더 자세히" → 그 항목만 깊이 분석 후 추가 보고
- "다른 티켓도 같이" → 추가 키 받아서 같은 흐름 반복
- "보완 액션 정리" → 누락·제안 항목을 *액션 리스트*로 재구성

## 실패 처리

- `tako show` 가 비-제로 종료 → stderr 그대로 노출 후 중단
- 인증 실패(401/403) → `tako init` 으로 자격 증명 갱신 안내
- 응답에 한국어 깨짐 → 그대로 노출, ADF 변환은 셸이 이미 처리한 상태라 추가 처리 불필요

## 민감 정보 주의

티켓 본문에 토큰·비밀번호·개인정보가 있을 수 있다. 본 검토는 *세션 안*에 그 본문 일부를 노출시키므로, 민감 티켓에는 이 커맨드를 신중히 사용한다 (v1.x 는 자동 필터링 없음).
